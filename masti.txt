MongoDB Export Concept (Important Notes)
1. What does module.exports do?

Whatever you assign to module.exports is what require() receives.

module.exports = main;

Then:

const main = require("./db");

Now main is just a function.

2. Why doesn't this work?
module.exports = main;

Then later:

main.collection("Data");

❌ Error

Because:

main is a function
.collection() belongs to a MongoDB database object
Functions don't have a .collection() method
3. Why can't we export collection directly?

Example:

let collection;

async function main() {
    await client.connect();
    collection = db.collection("Data");
}

module.exports = collection;
What happens?
Node starts reading file
collection = undefined
module.exports = collection
undefined gets exported
MongoDB connects later
Collection gets value

But export already happened.

Result:

const collection = require("./db");

receives:

undefined
Most Important Point

module.exports runs immediately.

MongoDB connection takes time (async).

So collection is not ready when export happens.

4. Why does async cause this?

Because:

await client.connect();

takes time.

Node doesn't wait for MongoDB before executing:

module.exports = collection;

So the export happens first.

5. Correct Thinking

Instead of exporting:

❌ main function

or

❌ collection directly

Export a function that returns the collection.

Example flow:

db.js
↓
Connect MongoDB
↓
Store collection
↓
Export getCollection()

Then:

index.js
↓
require("./db")
↓
Call getCollection()
↓
Receive collection
↓
insertOne()
6. Three Things You Can Export
Option 1
module.exports = main;

Exports:

main()

Just the function.

Option 2
module.exports = db;

Exports database object.

Then:

db.collection("Data")

can be used.

Option 3 (Most Common for Small Projects)
module.exports = getCollection;

Then:

const getCollection = require("./db");

const collection = getCollection();

Use:

await collection.insertOne(data);

✅ Simple and clean.

Interview-Level Understanding

Always ask yourself:

Question 1

When is this variable created?

Example:

collection

Is it available immediately or after connection?

Question 2

When does module.exports run?

Answer:

Immediately when Node loads the file.

Question 3

Is the database connection synchronous or asynchronous?

Answer:

Asynchronous.

One-Line Memory Trick
Exporting a database collection directly often fails because:

"The collection is created after the MongoDB connection, but module.exports runs before the connection finishes."

If you remember only one sentence from this entire discussion, remember that one. 🚀

CORS (Cross-Origin Resource Sharing) Notes
What is CORS?

CORS is a browser security feature that prevents a website from making requests to a different origin unless the server explicitly allows it.

Example:

Frontend:
http://127.0.0.1:3001

Backend:
http://localhost:3000

These are considered different origins, so the browser checks whether the backend allows the request.

Problem I Faced

When I used:

fetch("http://localhost:3000/register")

the browser showed:

No 'Access-Control-Allow-Origin' header is present

and blocked the request.

Reason:

Frontend Origin ≠ Backend Origin

and the server was not allowing cross-origin requests.

Why Postman Worked

Postman is not a browser.

Postman
↓
Send Request
↓
Receive Response

Postman does not enforce CORS rules.

Therefore:

Postman → Backend ✅

Browser → Backend ❌

until CORS is enabled.

Solution

Install CORS package:

npm i cors

Import it:

const cors = require("cors");

Use it in Express:

app.use(cors());

Place it before routes:

app.use(cors());
app.use(express.json());
app.use(router);
What app.use(cors()) Does

It adds headers to the response such as:

Access-Control-Allow-Origin: *

This tells the browser:

"This server allows requests from other origins."

After that:

Frontend
↓
fetch()
↓
Express
↓
MongoDB

works successfully.

Interview Answer

What is CORS?

CORS (Cross-Origin Resource Sharing) is a browser security mechanism that restricts web pages from making requests to a different origin unless the server explicitly allows those requests through CORS headers.

Key Point to Remember
Browser = Enforces CORS

Postman = Does Not Enforce CORS
